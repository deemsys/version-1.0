Администрирование

Внимание: Для того чтобы совершать все эти действия, вам должна быть назначена роль с соответствующими полномочиями. Для того чтобы определить, какая у вас роль, перейдите на страницу Роли участников. Основные сведения о ролях и правах доступа изложены также в Права доступа и роли: обзор.

Пользователи с правами доступа администратора могут изменять некоторые настройки поисковой машины на сайте. Эти пользователи имеют доступ к странице Администраторских настроек инстурмента "Поиск". Страница дает пользователям доступ к настройкам поисковой машины, которая используется инструментом "Поиск".

Администраторские настройки показывает информацию об индексировании рабочих потоков на сервере и их состонии. Она также показывает информацию, связанную с текущей индексацией.

Очередность индексов : Очередность индексов - это уровень индексирующих команд, которые управляют поисковыми машинами. Эти команды принимают 3 формы:

  • Глобальные команды, который управляют всей поисковой машиной.
  • Команды уровня сайта, которые управляют поисковой машиной в предеах сайта.
  • Команды уровня документа, которые информарую поисковую машину о документах, добавленных или удаленных из индекса.

Глобальные команды (Перестроить весь индекс, Обновить весь индекс, Снять блокировку). Только постоянный администратор может использовать Глобальные команды. Эти команды включают: перестроить индекс на сайте, обновить индекс на сайте и снять блокировку. Глобальная перестроение удаляет индекс целиком и запрашивает у всех инструмента системы отпраку своих материалов в поисковую машину для переиндексации. Эта операция должна производиться с осторожностью, так как большой сайт может отнять несколько часов на эту операцию. Глобальное обновление индекса более удобная команда, так как она сохраняет весь известный поисковой машине материал и обновляет индекс. Пока происходит индексирование, поисковая машина будет все еще содержать весь материал. Команда "снять блокировку" может использоваться оператором только в том случае, если поток данных прекратил свое существование. Эта опция практически не используется, так как поток данных достаточно стабилен. Если поток данных, тем не менее, прекратил свое существование, будет необходимо снять блокировку других потоков данных, чтобы рязрешить им продолжить начатую операцию.

Команды уровня сайта (Обновить индекс на сайте, Перестроить индекс на сайте). Администраторы или владельца сайта могут использовать опции перестроения ии обновления индекса своего сайта.

Перезагрузить индекс: Эта команда просто обновляет индекс, чтобы предоставить последнюю информацию о поисковой машине.

Информация об индексеInformation Feedback

Первый блок инормации содержит данные, когда индекс был загружен в послений раз на кластерном узле, к которому вы подсоединены. Она также информирует о текущей активности индекса, это значение может быть none, если индексации не производится, или ID потока данных индекса и ожидаемое время завершения, если производятся операции с индексом. Последний информационный элемент блока показывает статус команды в очереди команд индекса.

Блок "Mater Control Records " информарует о статусе любых Глобальных команд в очереди команд индекса. Блок " Site Control Records " информарует о статусе команд уровня сайта в очереди команд индекса. Эту будет включать также команды с других сайтов, касающиеся данного сайта

Блок "Indexer Workers" представляет список всех потоков дагнных в индексе в кластере sakai. В этом списке указаны id потока данных, время последнего обновления, статус потока даных. Первый из ID потоков данных представлен в первом блоке вместе в предполагаемым временем выволнения.

Наконец, существует информационный блок о сегментах, используемых поисковой машиной для сохранения информации.
